# AI 小说转剧本工具

一款通用的 AI 辅助剧本创作工具。它可以把 3 个章节以上的小说文本转换为结构化剧本 YAML，让作者快速得到可编辑、可继续打磨的剧本初稿。工具不绑定任何固定作品，用户可以粘贴或导入自己下载、整理的小说文本进行测试。

## 核心功能

- 支持粘贴小说文本，也支持导入 TXT/Markdown 文件。
- 支持一次选择多个 TXT/Markdown 文件，按文件名自然排序后合并为多章节输入。
- 支持 `第一章`、`Chapter 1`、`1. 标题` 等常见章节格式。
- 自动识别章节，并按章节生成幕、场景和节拍。
- 从对白和叙述中提取角色、地点、时间、情绪等剧本要素。
- 输出符合项目定义的 YAML 剧本结构。
- 提供转换统计、警告提示、复制 YAML、下载 YAML。
- 提供输入分析面板，转换前可查看章节数、字数、段落数、对白数和格式风险。
- 输入少于 3 章时会明确提示不满足题目要求。
- 支持可选 AI 增强：未配置 API Key 时使用规则引擎，配置后可让模型在同一 Schema 内优化场景、节拍和对白。
- 保留 AI 扩展入口，后续可接入大语言模型进行更准确的人物识别、场景拆分和对白改写。

## 快速启动

本项目后端使用 Python 标准库实现，不需要安装第三方依赖，只需要 Python 3.11 或更高版本。前端是静态页面，由 Python 服务直接托管。

```bash
npm start
```

启动后访问：

```text
http://localhost:4173
```

运行基础检查：

```bash
npm run check
```

运行通用转换测试：

```bash
npm test
```

## 使用方式

1. 输入作品名、主要角色、主题和小说正文，或点击“导入”选择一个或多个 TXT/Markdown 文件。
2. 小说正文建议使用 `第一章`、`第二章`、`第三章`、`Chapter 1` 或 `1. 标题` 等章节标题。
3. 导入或粘贴后，可点击“分析”检查章节识别和输入质量。
4. 选择改编模式、输出密度和转换引擎。
5. 点击“转换”，右侧会生成 YAML 剧本。
6. 点击复制或下载，保存剧本初稿。

## 可选 AI 增强

默认情况下，项目使用本地规则引擎，不依赖外部服务，便于比赛演示和评委复现。

如需启用 AI 增强，在启动服务前设置环境变量：

```powershell
$env:OPENAI_API_KEY="你的 API Key"
$env:OPENAI_MODEL="gpt-4.1-mini"
npm start
```

也支持 OpenAI-compatible 服务：

```powershell
$env:OPENAI_API_KEY="你的 API Key"
$env:OPENAI_BASE_URL="https://api.openai.com/v1"
$env:OPENAI_MODEL="gpt-4.1-mini"
npm start
```

AI 增强的设计原则：

- 规则引擎先生成稳定初稿，确保无 Key 时也可用。
- AI 只在既定 YAML Schema 内增强，不改变输出格式。
- AI 请求失败时自动回退到规则引擎，并在页面中显示原因。
- 密钥只从服务端环境变量读取，不会返回给浏览器。

## 当前实现说明

当前版本采用“Python 规则引擎 + 可选 AI adapter”的实现方式：

- `app/converter.py` 负责章节切分、对白识别、角色候选提取、场景拆分、节拍生成和 YAML 序列化。
- `app/quality.py` 负责结构校验、质量评分和修改建议。
- `app/ai_adapter.py` 负责可选 AI 增强和失败回退。
- `app/server.py` 使用 Python 标准库提供静态页面和 API。
- 这样可以保证没有外部 API 的情况下也能稳定演示和复现。
- AI adapter 会把 Python 规则结果作为初稿，再由模型进行二次润色、合并场景、补充人物弧光和镜头建议。
